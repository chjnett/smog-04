## 🚨 반드시 수정해야 할 것 (Critical)

### 1. Admin 보안 — 인증 없음
- `/admin` 페이지에 **인증/로그인이 없어** 누구나 접근 가능
- Supabase Auth 또는 비밀번호 보호 미들웨어 필요

### 2. 장바구니 버튼 미구현
- `app/shop/[id]/page.tsx` — "장바구니" 버튼이 있지만 **아무 동작도 하지 않음**
- 기능이 없다면 버튼 제거 또는 카카오톡 문의로 통합

### 3. 검색 버튼 미구현
- `components/site-header.tsx` — 헤더 검색 아이콘 클릭 시 아무 동작 없음

### 4. TypeScript 에러 무시 설정
- `next.config.mjs` — `ignoreBuildErrors: true` 상태
- 배포 전에 `false`로 바꾸고 타입 에러 수정 권장

### 5. 이미지 최적화 비활성화
- `images: { unoptimized: true }` — Next.js 이미지 최적화 꺼져 있음
- Supabase Storage 도메인을 `remotePatterns`에 추가하고 최적화 활성화 필요

## ⚠️ 수정 권장 (Important)

### 6. 데이터 없을 때 빈 화면
- 블로그/공지/리뷰 페이지 — 데이터가 0건이면 **빈 화면**만 표시
- "등록된 항목이 없습니다" 같은 empty state UI 필요

### 7. 블로그 key 값 문제
- `app/blog/page.tsx` — `key={idx}` 사용 → `key={post.id}` 사용해야 함
- notice, reviews 페이지도 동일 문제

### 8. 에러 핸들링 부족
- 모든 프론트 페이지에서 Supabase 에러 발생 시 **사용자에게 표시 안 됨**
- 네트워크 에러나 DB 에러 시 에러 UI 표시 필요

### 9. Footer 없음
- 전체 사이트에 **하단 푸터가 없음** (회사 정보, 사업자번호, 고객센터 등)
- 이커머스 사이트라면 법적 정보 표시가 필요할 수 있음

### 10. site-header.tsx 끝에 불필요한 경로
- 87번 줄에 `///Users/cheonhyeonjun/...` 디버그 경로가 남아있음

### 15. 페이지별 metadata 부족
