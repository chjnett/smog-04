이 문서는 Dint 쇼핑몰에서 상품을 어떻게 등록하고, 왜 간혹 메인 페이지에서 보이지 않는지에 대해 설명합니다.

## 1. 상품 등록 로직 (Admin)

관리자 페이지([admin/page.tsx](file:///Users/cheonhyeonjun/workspace/luxury-e-commerce-app/app/admin/page.tsx))에서 상품을 등록할 때 수행되는 단계는 다음과 같습니다.

1.  **이미지 업로드**: 선택한 이미지를 Supabase Storage의 `product-images` 버킷에 저장하고 공개 URL을 가져옵니다.
2.  **데이터 구성**: 입력된 명칭, 브랜드, 가격, 카테고리 등을 객체로 만듭니다.
3.  **DB 저장**: Supabase의 `products` 테이블에 `insert` 합니다. 이때 `id`와 `created_at`은 DB에서 자동으로 생성됩니다.

## 2. 메인 페이지 노출 원리 (Home)

메인 페이지([app/page.tsx](file:///Users/cheonhyeonjun/workspace/luxury-e-commerce-app/app/page.tsx))에서 상품을 보여주는 방식은 다음과 같습니다.

1.  **전체 상품 조회**: 최신순(`created_at DESC`)으로 모든 상품을 가져옵니다.
2.  **필터링**: 현재 선택된 카테고리나 검색어에 맞는 상품만 걸러냅니다.
3.  **갯수 제한 (중요)**: 
    *   메인 페이지의 '전체' 보기 상태에서는 **최상위 8개 상품**만 보여주도록 설계되어 있습니다.
    *   `filteredProducts.slice(0, 8)` 코드로 인해 9번째 이후의 상품은 메인에서 사라지고 `/shop` 페이지에서만 보이게 됩니다.

## 3. "상품이 안 보여요" 문제 해결 (Debugging)

상품을 등록했는데 메인에서 안 보인다면 다음 항목들을 순서대로 확인해 보세요.

### ✅ 체크리스트 1: 8개 제한 확인
*   현재 등록된 총 상품이 8개가 넘었나요? 
*   새로 등록한 상품이 '최신순' 정렬에서 밀려나 9번째 이후가 되면 메인 페이지에서는 보이지 않습니다.
*   **확인 방법**: 하단의 `View All Products` 버튼을 눌러 `/shop` 페이지로 이동했을 때 상품이 보이는지 확인하세요.

### ✅ 체크리스트 2: 카테고리 매칭 (텍스트 일치)
*   등록할 때 입력한 카테고리명이 정확한가요?
*   시스템은 **텍스트 기반 매칭**(`isMatch`)을 사용합니다. 만약 카테고리 명칭이 DB와 미세하게 다르면 필터링 과정에서 제외될 수 있습니다.

### ✅ 체크리스트 3: 데이터 동기화 (새로고침)
*   Admin에서 등록 완료 후 메인 페이지를 **새로고침** 하셨나요?
*   현재 페이지는 실시간 소켓 통신(Realtime)이 아닌, 페이지 로드 시 데이터를 한 번 가져오는 방식입니다.

```typescript
// app/page.tsx
if (productsRes.data) {
  console.log("가져온 모든 상품:", productsRes.data);

}
```

브라우저 개발자 도구(F12)의 **Console** 탭을 확인하면 실제로 DB에서 데이터를 가져왔는지, 필터링에서 걸러졌는지 바로 알 수 있습니다.
